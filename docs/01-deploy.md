# 1. Запуск сайта

Здесь – как из папки с кодом сделать работающий сайт в интернете. Делается один
раз. Дальше обновлять сайт будет легко (см. конец раздела).

Порядок такой:

1. Загрузить код на **GitHub**.
2. Создать базу данных в **Supabase**.
3. Развернуть сайт на **Vercel** и вписать ключи.
4. Создать первого администратора (тебя).
5. Подключить домен ki-school.

> Совет: открой каждый сервис в отдельной вкладке браузера – так удобнее
> переключаться.

---

## Шаг 1. Загрузить код на GitHub

GitHub – это «облако» для кода. Самый простой способ без терминала – программа
**GitHub Desktop**.

1. Зарегистрируйся на [github.com](https://github.com) (бесплатно).
2. Скачай и установи **GitHub Desktop**: [desktop.github.com](https://desktop.github.com).
   Войди в нём под своим аккаунтом GitHub.
3. В GitHub Desktop: меню **File → Add local repository** → выбери папку проекта
   `ki-school` → если предложит «create a repository», согласись (**Create a
   repository**).
4. Нажми **Publish repository**. Галочку «Keep this code private» можно оставить
   включённой (код будет приватным). Нажми **Publish**.

Готово – код теперь на GitHub. Папки `node_modules` и секретные файлы туда не
попадут (так настроено заранее).

---

## Шаг 2. Создать базу данных (Supabase)

1. Зарегистрируйся на [supabase.com](https://supabase.com) (можно войти через
   GitHub).
2. Нажми **New project**. Придумай название (например, `ki-school`), задай
   **Database Password** (запиши его себе) и выбери регион поближе (например,
   **Frankfurt**). Нажми **Create new project** и подожди пару минут.
3. Когда проект создан, открой слева **SQL Editor** → **New query**.
   - Открой в папке проекта файл `supabase/migrations/0001_init.sql`, скопируй
     весь его текст, вставь в окно запроса и нажми **Run** (или Ctrl/Cmd+Enter).
   - Повтори то же самое с файлом `supabase/migrations/0002_admin_content.sql`.
   - Должно написать «Success». Это создало все таблицы.
4. Теперь возьми ключи. Открой слева **Project Settings** (шестерёнка) → **API**.
   Понадобятся три значения (скопируй их в блокнот, скоро вставим в Vercel):
   - **Project URL** – длинный адрес вида `https://xxxx.supabase.co`
   - **anon public** – длинный ключ (он публичный, не страшно)
   - **service_role** – **секретный** ключ (никому не показывай!)

---

## Шаг 3. Развернуть сайт на Vercel

1. Зарегистрируйся на [vercel.com](https://vercel.com) через **GitHub** (так
   проще всего).
2. Нажми **Add New… → Project**. Vercel покажет твои репозитории GitHub – выбери
   `ki-school` и нажми **Import**.
3. На экране настроек **не меняй** Build-настройки (Vercel сам поймёт, что это
   Next.js). Прежде чем нажимать Deploy, раскрой **Environment Variables** и
   добавь три переменные (имя слева, значение справа):

   | Имя (Name)                      | Значение (Value)                  |
   | ------------------------------- | --------------------------------- |
   | `NEXT_PUBLIC_SUPABASE_URL`      | Project URL из Supabase           |
   | `NEXT_PUBLIC_SUPABASE_ANON_KEY` | ключ **anon public**              |
   | `SUPABASE_SERVICE_ROLE_KEY`     | ключ **service_role** (секретный) |

   Добавляй по одной: вписал имя и значение → **Add**.
4. Нажми **Deploy** и подожди 1–2 минуты. В конце увидишь поздравление и кнопку
   на сайт – он уже в интернете по адресу вида `ki-school-xxxx.vercel.app`.

> Если добавил переменные уже после первого деплоя – зайди в **Settings →
> Environment Variables**, добавь их, затем **Deployments → … → Redeploy**.

---

## Шаг 4. Создать первого администратора (тебя)

Родителей и учеников ты будешь создавать прямо в админке. Но самого первого
администратора нужно завести вручную – это делается один раз, через браузер.

1. В Supabase открой слева **Authentication → Users → Add user → Create new
   user**.
2. Заполни:
   - **Email:** `admin@parents.ki-school.local` (именно так, латиницей)
   - **Password:** придумай надёжный пароль (это твой пароль от админки)
   - включи галочку **Auto Confirm User**
   - нажми **Create user**.
3. Теперь сделай этого пользователя администратором. Открой слева **SQL Editor →
   New query**, вставь и запусти:

   ```sql
   update public.profiles
   set username = 'admin', is_admin = true
   where id = (select id from auth.users
               where email = 'admin@parents.ki-school.local');
   ```

Готово. Теперь на сайте на странице **Войти** введи логин `admin` и свой пароль –
откроется админ-панель `/admin`.

> Как пользоваться админкой – в разделе [2. Админ-панель](02-admin.md).

---

## Шаг 5. Подключить домен ki-school

1. Купи домен (например, `ki-school.ru`) у любого регистратора (Reg.ru, Timeweb,
   Namecheap и т.п.).
2. В Vercel открой проект → **Settings → Domains** → впиши свой домен (например
   `ki-school.ru`) → **Add**.
3. Vercel покажет, какие записи прописать у регистратора (обычно запись типа
   **A** на IP `76.76.21.21` и/или **CNAME** для `www`). Зайди в личный кабинет
   регистратора, в раздел управления DNS, и добавь то, что показал Vercel.
4. Подожди (от нескольких минут до пары часов). Когда напротив домена в Vercel
   загорится «Valid», сайт открывается по твоему адресу. HTTPS (замочек) Vercel
   включает автоматически.

---

## Как обновлять сайт дальше

Любое изменение (новая тема, новая лаба, правка кода) попадает на сайт так:

1. Меняешь файлы – либо через **GitHub Desktop** (Commit → Push), либо прямо на
   сайте **github.com** (см. раздел [3. Материалы](03-materials.md)).
2. Vercel сам замечает изменение и за ~1 минуту обновляет сайт. Ничего больше
   делать не нужно.

А тексты, услуги, отзывы, прогресс учеников, даты и заявки меняются вообще без
кода – прямо в **админ-панели** на сайте.

---

## Если что-то пошло не так

- **На странице «Войти» пишет, что база не подключена.** Значит, в Vercel не
  заданы переменные окружения (Шаг 3) – добавь их и сделай Redeploy.
- **Не могу войти под admin.** Проверь, что email пользователя ровно
  `admin@parents.ki-school.local` и что выполнил SQL-запрос из Шага 4
  (логин `admin`, пароль – тот, что задал при создании пользователя).
- **Сайт не открывается по домену.** DNS-записи обновляются не мгновенно;
  подожди и проверь, что записи у регистратора совпадают с тем, что показал
  Vercel.
- **После изменения в админке на сайте всё по-старому.** Обнови страницу
  (Ctrl/Cmd+R). Изменения применяются сразу.
- **Сайт «уснул» / медленно открылся первый раз.** На бесплатном тарифе
  Supabase база может «засыпать» при простое и просыпается за несколько секунд –
  это нормально.
